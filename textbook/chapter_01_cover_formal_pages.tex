\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
\label{chap:abstract}

The Faculty of Engineering at Beni-Suef University (BSU) has historically relied on fragmented, Excel-based workflows for academic management---student registration, grade tracking, course assignments, and administrative reporting. This manual approach created data inconsistency across departments, lacked audit trails for administrative actions, and scaled poorly as enrollment exceeded 4,000 active students.

\textbf{Release BSU Engineering Portal} addresses these challenges through a modern, decoupled web architecture comprising: (1) a \textbf{Django 5.0} backend with Django REST Framework providing secure, stateless API endpoints with cross-origin Session/CSRF protection; (2) a \textbf{React 19} frontend built via Vite, utilizing Material-UI for responsive interfaces; (3) \textbf{MySQL 8.0} as the relational database ensuring strict ACID compliance for grade and attendance transactions; and (4) \textbf{Docker} containerization enabling deterministic, portable deployments across any cloud or on-premises infrastructure.

Key system capabilities include hierarchical academic entity modeling (departments $\rightarrow$ specializations $\rightarrow$ levels $\rightarrow$ students), transaction-safe batch CSV/Excel import pipelines using Pandas for student enrollment and grade uploads, comprehensive audit logging for all administrative actions, and five distinct user roles (Admin, Doctor, Student, Student Affairs, Staff Affairs) strictly enforced via Role-Based Access Control (RBAC).

The project demonstrates engineering maturity through robust DevOps practices, security-hardened deployment (configured CORS/CSRF boundaries, input validation, rate limiting), and explicitly defined system scope, providing a highly scalable foundation for Beni-Suef University's ongoing digital transformation.

\vspace{1cm}
\noindent\textbf{Keywords:} Academic ERP, Django REST Framework, React, Role-Based Access Control (RBAC), Docker, ACID Transactions, Bulk Data Processing, DevOps
